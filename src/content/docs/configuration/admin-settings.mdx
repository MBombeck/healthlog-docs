HealthLog stores its configuration in an **AppSettings singleton** in the database. These settings are managed through the admin interface at `/settings/admin` and take effect immediately — no restart required.

## General

| Setting | Description | Default |
|---|---|---|
| `registrationEnabled` | Enable or disable registration for new users. The first user can **always** register, regardless of this setting. | `true` |
| `defaultLocale` | Default language for new users: `de` or `en` | `de` |

:::caution
Registration is **enabled** by default. Disable it after creating all desired accounts to prevent unauthorized signups.
:::

## Notifications

Global toggles disable the respective channel for **all** users. Individual user preferences are overridden.

| Setting | Description | Default |
|---|---|---|
| `telegramGlobal` | Enable Telegram notifications globally | `true` |
| `ntfyGlobal` | Enable ntfy notifications globally | `true` |
| `webPushGlobal` | Enable Web Push notifications globally | `true` |

## Features

| Setting | Description | Default |
|---|---|---|
| `apiGlobal` | Enable or disable the external API | `true` |
| `moodLogGlobal` | Enable MoodLog integration (mood sync) | `true` |

## Reminders

| Setting | Description | Default |
|---|---|---|
| `reminderLateMinutes` | Minutes after a scheduled intake before a reminder is considered "late" | `120` |
| `reminderMissedMinutes` | Minutes after a scheduled intake before a reminder is marked as "missed" | `240` |

## Analytics (Umami)

| Setting | Description |
|---|---|
| `umamiEnabled` | Enable Umami tracking |
| `umamiScriptUrl` | URL to the Umami tracking script |
| `umamiWebsiteId` | Umami website ID |

## Error Tracking (GlitchTip)

| Setting | Description |
|---|---|
| `glitchtipEnabled` | Enable GlitchTip/Sentry error tracking |
| `glitchtipDsn` | Sentry-compatible DSN URL |
| `glitchtipEnvironment` | Environment name (e.g. `production`, `staging`) |

## Web Push (VAPID)

VAPID keys can be set directly in the admin settings. These take precedence over the corresponding environment variables.

| Setting | Description |
|---|---|
| `vapidPublicKey` | VAPID public key |
| `vapidPrivateKey` | VAPID private key (stored encrypted) |
| `vapidSubject` | Contact URI for VAPID (`mailto:...`) |

## Bug Reports (GitHub)

Allows users to submit bug reports as GitHub issues directly from within the app.

| Setting | Description |
|---|---|
| `githubIssueToken` | GitHub Personal Access Token (stored encrypted) |
| `githubIssueRepo` | Target repository in `owner/repo` format |

:::note
Sensitive values like the VAPID private key and GitHub token are stored encrypted in the database using AES-256-GCM. The encryption key is the `ENCRYPTION_KEY` environment variable.
:::
