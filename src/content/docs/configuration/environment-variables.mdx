HealthLog is configured through environment variables. Place them in a `.env` file next to your `docker-compose.yml`.

## Required

These variables **must** be set for HealthLog to start.

| Variable | Description | Example |
|---|---|---|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://healthlog:healthlog@db:5432/healthlog` |
| `SESSION_SECRET` | Used to sign sessions, 64 hex characters | `openssl rand -hex 32` |
| `ENCRYPTION_KEY` | AES-256-GCM key, 64 hex characters | `openssl rand -hex 32` |
| `API_TOKEN_HMAC_KEY` | HMAC key for API token hashing, 64 hex characters | `openssl rand -hex 32` |

:::danger
`ENCRYPTION_KEY` must **never** be changed after the initial startup. All encrypted database fields (tokens, OAuth credentials, API keys) are encrypted and decrypted with this key. Changing it will make all encrypted data permanently unrecoverable.
:::

## Important

| Variable | Description | Example |
|---|---|---|
| `NEXT_PUBLIC_APP_URL` | Public URL of the app. Used for Passkey RP-ID, OAuth redirects, and webhooks. | `https://health.example.com` |
| `APP_URL` | Alternative name for `NEXT_PUBLIC_APP_URL` (fallback) | `https://health.example.com` |
| `NODE_ENV` | `production` or `development` | `production` |

## Telegram

| Variable | Description |
|---|---|
| `TELEGRAM_WEBHOOK_SECRET` | Secret string used to verify incoming Telegram webhooks |

## Withings

| Variable | Description |
|---|---|
| `WITHINGS_REDIRECT_URI` | Optional override for the OAuth redirect URI |
| `WITHINGS_WEBHOOK_SECRET` | Secret used to verify incoming Withings webhooks |

## Logging (Loki)

| Variable | Description | Default |
|---|---|---|
| `LOG_LEVEL` | Minimum log level: `debug`, `info`, `warn`, `error` | `info` |
| `LOKI_ENDPOINT` | Loki push URL, e.g. `https://loki.example.com/loki/api/v1/push` | — |
| `LOKI_USERNAME` | Basic auth username for Loki | — |
| `LOKI_PASSWORD` | Basic auth password for Loki | — |
| `LOG_SAMPLE_RATE` | Log sampling rate (`0.0`–`1.0`) | `1.0` |
| `LOG_SLOW_THRESHOLD_MS` | Threshold for slow request warnings in ms | `3000` |
| `LOG_INCLUDE_STACK` | Include stack traces in error logs | `true` |

## Monitoring

| Variable | Description |
|---|---|
| `COMMIT_SHA` / `GIT_COMMIT` / `SOURCE_COMMIT` | Git commit hash of the build (used for health checks and error tracking) |
| `BUILD_TIMESTAMP` | Build timestamp |

## Web Push (Fallback)

These variables are optional — Web Push can also be configured through the admin settings. Environment variables serve as a fallback.

| Variable | Description |
|---|---|
| `VAPID_PUBLIC_KEY` / `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | VAPID public key for Web Push |
| `VAPID_PRIVATE_KEY` | VAPID private key |
| `VAPID_SUBJECT` / `VAPID_EMAIL` | Contact URI or email for VAPID (`mailto:admin@example.com`) |

## Bug Reports

| Variable | Description |
|---|---|
| `GITHUB_ISSUE_TOKEN` | GitHub Personal Access Token with `repo` scope |
| `GITHUB_ISSUE_REPO` | Target repository in `owner/repo` format |

## .env.example

```bash
# === Required ===
DATABASE_URL=postgresql://healthlog:healthlog@db:5432/healthlog
SESSION_SECRET=          # openssl rand -hex 32
ENCRYPTION_KEY=          # openssl rand -hex 32 — NEVER change after first start!
API_TOKEN_HMAC_KEY=      # openssl rand -hex 32

# === App ===
NEXT_PUBLIC_APP_URL=https://health.example.com
NODE_ENV=production

# === Telegram ===
TELEGRAM_WEBHOOK_SECRET=

# === Withings ===
WITHINGS_REDIRECT_URI=
WITHINGS_WEBHOOK_SECRET=

# === Logging ===
LOG_LEVEL=info
LOKI_ENDPOINT=
LOKI_USERNAME=
LOKI_PASSWORD=

# === Monitoring ===
COMMIT_SHA=
BUILD_TIMESTAMP=

# === Web Push (optional, falls back to admin settings) ===
VAPID_PUBLIC_KEY=
VAPID_PRIVATE_KEY=
VAPID_SUBJECT=

# === Bug Reports ===
GITHUB_ISSUE_TOKEN=
GITHUB_ISSUE_REPO=
```
