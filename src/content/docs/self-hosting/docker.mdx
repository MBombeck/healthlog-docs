HealthLog is distributed as a Docker image and can be deployed with Docker Compose in just a few minutes.

## Docker Compose

Create a `docker-compose.yml`:

```yaml
services:
  app:
    image: ghcr.io/mbombeck/healthlog:latest
    container_name: healthlog
    restart: unless-stopped
    ports:
      - "3000:3000"
    env_file:
      - .env
    depends_on:
      db:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

  db:
    image: postgres:16-alpine
    container_name: healthlog-db
    restart: unless-stopped
    environment:
      POSTGRES_USER: healthlog
      POSTGRES_PASSWORD: changeme
      POSTGRES_DB: healthlog
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "healthlog"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  pgdata:
```

:::danger
Replace `POSTGRES_PASSWORD: changeme` with a secure, randomly generated password **before the first start** (`openssl rand -base64 24`). The same password must be set in `DATABASE_URL` in your `.env` file. Leaving the default password unchanged is a serious security risk.
:::

## Environment Variables

Create a `.env` file in the same directory:

```bash
# Database
DATABASE_URL=postgresql://healthlog:changeme@db:5432/healthlog

# App
NEXT_PUBLIC_APP_URL=https://health.example.com
NODE_ENV=production

# Cryptographic keys (generate each with: openssl rand -hex 32)
SESSION_SECRET=
ENCRYPTION_KEY=
API_TOKEN_HMAC_KEY=

# Disable telemetry
NEXT_TELEMETRY_DISABLED=1
```

See [Security -- Self-Hosting](/security/self-hosting/) for details on the cryptographic keys.

## Starting Up

```bash
docker compose up -d
```

## Startup Process

The Docker image uses a `docker-entrypoint.sh` that automatically performs the following steps on startup:

1. **Waits for PostgreSQL** -- up to 30 connection attempts with a 1-second delay
2. **Runs database migrations** -- `prisma migrate deploy`
3. **Starts the Node.js server** -- `node server.js`

No manual migration step is needed.

## Docker Image Details

| Property | Value |
|----------|-------|
| Base image | Node 22 Alpine |
| Build | Multi-stage (builder + runner) |
| User | `nextjs` (UID 1001, non-root) |
| Port | 3000 |
| Healthcheck | `GET /api/health` every 30s |

The non-root user ensures that even in the event of a container breakout, no root privileges are gained on the host.

## Accessing HealthLog

After startup, HealthLog is available at `http://localhost:3000`. For production use, you should place a [Reverse Proxy](/self-hosting/reverse-proxy/) with HTTPS in front of it.

:::caution
Port 3000 should **not** be directly exposed to the internet. Only expose the reverse proxy (ports 443/80). If Docker maps the port on the host, make sure a firewall blocks external access, or remove the `ports` mapping and let the reverse proxy connect through the Docker network.
:::
