In production, HealthLog should run behind a reverse proxy with TLS termination. HTTPS is required -- session cookies are set to `secure: true` and passkeys need a secure connection.

Below are configuration examples for the most common reverse proxies.

<Tabs>
  <TabItem label="Traefik">

Add Traefik labels to your `docker-compose.yml`:

```yaml
services:
  app:
    image: ghcr.io/mbombeck/healthlog:latest
    container_name: healthlog
    restart: unless-stopped
    env_file:
      - .env
    depends_on:
      db:
        condition: service_healthy
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.healthlog.rule=Host(`health.example.com`)"
      - "traefik.http.routers.healthlog.entrypoints=websecure"
      - "traefik.http.routers.healthlog.tls.certresolver=letsencrypt"
      - "traefik.http.services.healthlog.loadbalancer.server.port=3000"
    networks:
      - traefik
      - default

networks:
  traefik:
    external: true
```

The port mapping (`ports: "3000:3000"`) is no longer needed since Traefik connects directly through the Docker network.

  </TabItem>
  <TabItem label="Caddy">

Add the following block to your `Caddyfile`:

```
health.example.com {
    reverse_proxy localhost:3000

    header {
        X-Content-Type-Options nosniff
        X-Frame-Options DENY
        Referrer-Policy strict-origin-when-cross-origin
    }
}
```

Caddy automatically handles Let's Encrypt certificates and HTTPS redirects.

  </TabItem>
  <TabItem label="nginx">

Create a server configuration at `/etc/nginx/sites-available/healthlog`:

```nginx
server {
    listen 443 ssl http2;
    server_name health.example.com;

    ssl_certificate /etc/letsencrypt/live/health.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/health.example.com/privkey.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;

        # WebSocket support
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # Forward headers
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name health.example.com;
    return 301 https://$host$request_uri;
}
```

Enable the configuration:

```bash
ln -s /etc/nginx/sites-available/healthlog /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

  </TabItem>
</Tabs>

## Important Notes

- **APP_URL** must contain the public domain (e.g. `https://health.example.com`). Passkey registration and OAuth redirects depend on it.
- **WebSocket support** is recommended for future features. The nginx configuration above already includes the necessary `Upgrade` directives.
- **Port 3000** should not be directly exposed to the internet. Only expose the reverse proxy (443/80).
