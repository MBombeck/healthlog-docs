Medication management is the core of HealthLog. It covers medication tracking, time-window-based schedules, a phased reminder system, and compliance analytics.

## Adding Medications

Each medication is defined with the following attributes:

- **Name** -- Free-text field
- **Dose** -- Default dose (e.g., "10mg")
- **Category** -- User-defined categories for grouping
- **Active/Inactive** -- Only active medications generate reminders

## Schedules

Schedules define when a medication should be taken.

```typescript
interface MedicationSchedule {
  windowStart: string;   // e.g. "08:00"
  windowEnd: string;     // e.g. "10:00"
  label?: string;        // e.g. "Morning", "Evening"
  daysOfWeek?: string;   // Comma-separated day numbers (0=Sun..6=Sat), null = daily
  dose?: string;         // Optional: overrides the default dose
}
```

- **Time window**: Defined by `windowStart` and `windowEnd`
- **Days of week**: Restrict intake to specific days
- **Dose override**: Each schedule can specify a different dose

:::tip
For medications with varying doses on different days (e.g., Methotrexate), simply create multiple schedules with different days and doses.
:::

## Intake Tracking

Intakes are recorded with idempotency protection: duplicate entries for the same schedule within 60 seconds are automatically prevented.

### Intake Sources

| Source | Description |
|--------|------------|
| `WEB` | Manually via the web interface |
| `API` | Via the external API (Bearer token) |
| `REMINDER` | Via inline buttons in notifications |
| `IMPORT` | From a JSON import |

## Phase-Based Reminders

The reminder system uses four phases, signaled both visually and via notifications:

| Phase | Color | Meaning |
|-------|-------|---------|
| GREEN | Green | Time window has started -- intake now possible |
| YELLOW | Yellow | Time window ending soon -- reminder |
| ORANGE | Orange | Time window exceeded -- overdue |
| RED | Red | Intake missed -- marked as missed |

### Phase Configuration

Phase transitions are configurable **per medication**:

- **Minutes** -- Fixed time value (e.g., 15 minutes before the window ends)
- **Percent** -- Percentage of the time window (e.g., 80% of the window elapsed)

:::note
The default configuration works well for most cases. Custom adjustments are particularly useful for very short or very long time windows.
:::

## Compliance Analytics

HealthLog calculates compliance rates at three levels:

- **7-day rate** -- Short-term adherence
- **30-day rate** -- Monthly compliance
- **90-day heatmap** -- Visual overview of adherence over three months

The heatmap uses color coding to show whether all doses were taken, partially taken, or missed for each day.

## Pause / Snooze

- **Pause**: Suspends all reminders for a medication (e.g., while consulting a doctor)
- **Snooze**: Postpones a single reminder by a configurable duration

## Categories

Medications can be organized into user-defined categories. Categories help keep the dashboard organized and serve as filters in the compliance analytics.
