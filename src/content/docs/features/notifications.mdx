HealthLog supports three notification channels that can run in parallel. The system follows an opt-out model: all event types are enabled by default.

## Channels

### Telegram

Each user provides their own bot token and chat ID. The bot token is stored AES-256-GCM encrypted in the database.

**Features:**
- Inline buttons in messages: Taken / Snooze / Skipped
- Automatic message deletion after 1 hour (prevents stale buttons)
- Bot commands: `/help`, `/start`, `/add`, `taken <Name>`

:::tip
The free-text command `taken Ibuprofen` lets you confirm an intake directly in the Telegram chat without opening the app.
:::

### ntfy

Self-hosted or public ntfy instance. Configurable per user:

- **Server URL** -- e.g., `https://ntfy.example.com`
- **Topic** -- e.g., `healthlog-marc`
- **Authentication** -- Optional (Basic Auth or token)
- **Priority** -- Set automatically based on event type

### Web Push

Browser-based push notifications via VAPID keys.

- VAPID keys are generated during setup
- Subscriptions are device-specific (multiple devices per user supported)
- Works even when the browser tab is closed

## Event Types

| Event Type | Description |
|-----------|------------|
| `MEDICATION_REMINDER` | Reminder for a due intake (phase-based) |
| `MEASUREMENT_ANOMALY` | Reading outside the normal range |
| `COMPLIANCE_LOW` | Adherence below threshold |
| `WITHINGS_SYNC_FAILED` | Withings sync failed |
| `SYSTEM_ALERT` | System notifications (backup, errors) |

## Configuration

### Opt-Out Model

By default, all event types are active on all channels. Users can selectively disable them:

- Per channel (e.g., Telegram on, ntfy off)
- Per event type per channel (e.g., `SYSTEM_ALERT` on ntfy only)

### Global Settings

Administrators can control notifications system-wide:

- Global toggles per channel
- Quiet hours window (no notifications at night)

:::caution
Global admin settings override individual user preferences. If a channel is globally disabled, no user will receive notifications through it.
:::
