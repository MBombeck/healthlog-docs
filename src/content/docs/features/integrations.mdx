HealthLog connects with external services and devices. All integrations are optional and configured per user.

## Withings

Integration with Withings scales and blood pressure monitors via OAuth 2.0.

### Supported Devices and Metrics

| Device Type | Metrics |
|------------|---------|
| Scale | Weight, body fat percentage |
| Blood Pressure Monitor | Systolic, diastolic, heart rate |

### Synchronization

Data is synced through two mechanisms:

- **Webhook** -- Real-time notification on new readings (primary)
- **Polling** -- Hourly fallback fetch (secondary, in case the webhook fails)

### Prerequisites

The Withings integration requires a **Withings Developer Account** with a registered app. The Client ID and Client Secret are entered in the HealthLog settings and stored AES-256-GCM encrypted.

### Setup

1. Create a Withings Developer App at [developer.withings.com](https://developer.withings.com)
2. Enter the Client ID and Client Secret in the HealthLog settings
3. Connect your Withings account in the HealthLog settings (OAuth 2.0 flow)
4. Grant permission for measurement data
5. The webhook is registered automatically

:::note
If the webhook stops working (e.g., after a server migration), the hourly polling kicks in automatically. The `WITHINGS_SYNC_FAILED` event type notifies you of prolonged outages.
:::

## MoodLog

Integration with [moodlog.app](https://moodlog.app) for mood tracking data.

- **Webhook**: MoodLog sends new entries to HealthLog via webhook
- **Polling**: Periodic fetch as a fallback
- **Data**: Mood entries with tags

Mood data feeds into the AI correlation analysis (mood vs. blood pressure, weight, heart rate).

## External API

HealthLog provides an API for external systems, such as automation tools or IoT devices.

### Bearer Token

Tokens are created per medication and use the `hlk_` prefix.

```bash
curl -X POST https://healthlog.example.com/api/ingest/medication \
  -H "Authorization: Bearer hlk_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"medicationName": "Metoprolol", "idempotencyKey": "unique-key-123"}'
```

**Properties:**
- Per-medication scope (a token can only log intakes for its assigned medication)
- Stored as a SHA-256 hash in the database
- Revocable through the UI

:::tip
The external API is great for NFC tags on your medicine cabinet: scan the tag, and the intake is logged automatically.
:::

## Bug Reports

Users can optionally submit bug reports directly as GitHub issues.

- Requires a GitHub Personal Access Token (PAT) configured by the administrator
- Issues are created in the configured repository
- System info is attached automatically (version, browser, anonymized error data)

:::caution
The GitHub PAT is configured by the administrator. Users do not need their own GitHub account to submit bug reports.
:::
