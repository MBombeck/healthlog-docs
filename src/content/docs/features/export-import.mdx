HealthLog provides comprehensive export and import capabilities for data backup, sharing with doctors, and migration.

## Export

### Formats

| Format | Description |
|--------|------------|
| **JSON** | Structured format, suitable for backups and re-import |
| **CSV** | Tabular format, compatible with Excel, Google Sheets, etc. |

### Export Types

Data can be exported individually or as a complete package:

- `measurements` -- All health readings
- `medications` -- Medications and schedules
- `intake` -- Intake log
- `mood` -- Mood entries (when MoodLog integration is active)
- `all` -- Full export of all data

### Rate Limit

```
Maximum exports: 10 per hour per user
```

:::note
The rate limit prevents accidental repeated exports and excessive load. It is more than sufficient for regular use.
:::

## Import

### Format

The import accepts JSON files with the following structure:

```json
{
  "measurements": [
    {
      "type": "WEIGHT",
      "value": 82.5,
      "measuredAt": "2026-03-01T08:00:00Z"
    }
  ],
  "moodEntries": [
    {
      "mood": 7,
      "tags": ["relaxed"],
      "createdAt": "2026-03-01T20:00:00Z"
    }
  ]
}
```

### Limitations

- **Maximum records**: 10,000 per import
- **Supported data**: `measurements` and `moodEntries`
- **Deduplication**: Duplicates are detected automatically via unique constraints and skipped

:::caution
Imports never overwrite existing data. Duplicates (same type, same timestamp) are silently ignored.
:::

## Automatic Backup

HealthLog creates a weekly encrypted backup:

- **Schedule**: Sundays at 03:00 (pg-boss job)
- **Encryption**: AES-256-GCM
- **Storage**: In the database (as an encrypted blob)
- **Content**: All user data (readings, medications, intakes, mood)

```
Schedule: Sunday 03:00 UTC
Encryption: AES-256-GCM
Storage: PostgreSQL (encrypted blob)
Job Runner: pg-boss
```

:::caution
The automatic in-app backup is stored **in the same database**. If the database is lost, these backups are lost too. Set up an external PostgreSQL backup as well (e.g., `pg_dump` via cron job). See [Self-Hosting Security](/security/self-hosting/) for details.
:::
