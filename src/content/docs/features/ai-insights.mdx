HealthLog offers AI-powered analysis of your health data. The integration uses OpenAI (gpt-4o-mini) with a BYOK (Bring Your Own Key) model.

## Setup

Users provide their own OpenAI API key in the settings. The key is stored AES-256-GCM encrypted in the database.

:::note
Without an API key, AI Insights are disabled. All other HealthLog features remain fully functional.
:::

## Privacy Modes

Users choose what data is sent to OpenAI:

| Mode | Description |
|------|------------|
| **Aggregated** | Only statistical summaries (average, min, max, trend) |
| **Raw** | Actual readings with timestamps |

:::caution
In Raw mode, real health data is sent to the OpenAI API. HealthLog never sends data to third parties unless the user has explicitly configured the OpenAI integration.
:::

## Insight Categories

HealthLog generates insights across seven categories:

1. **General** -- Overall health trends
2. **Blood Pressure** -- Trend analysis, classification, recommendations
3. **Weight** -- Trend analysis, BMI development
4. **Heart Rate** -- Resting heart rate trends, anomalies
5. **BMI** -- Long-term development, target range
6. **Medication Compliance** -- Adherence analysis, patterns
7. **Mood** -- Mood trends and correlations (when MoodLog integration is active)

## Correlation Analysis

The AI identifies relationships between different metrics:

- Weight and blood pressure
- Mood and blood pressure (when MoodLog integration is active)
- Mood and weight
- Mood and heart rate
- Blood pressure and medication compliance

## Caching and Generation

- **Cache duration**: 24 hours per insight
- **Automatic generation**: Nightly between 02:00 and 02:30 via pg-boss job
- **Manual generation**: Available anytime from the dashboard (invalidates the cache)

```
Schedule: 02:00-02:30 UTC (pg-boss cron)
Cache TTL: 24 hours
Model: gpt-4o-mini
```

:::tip
The nightly auto-generation ensures that fresh insights are ready when you open the app in the morning, with no waiting required.
:::
