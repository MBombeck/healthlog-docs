## Registration

The first registered user is automatically assigned the `ADMIN` role.

```
POST /api/auth/register
```

**Rate Limit:** 5 requests / 15 minutes / IP

**Request Body:**

```json
{
  "username": "marc",
  "email": "marc@example.com",
  "password": "sicheres-passwort"
}
```

**Response** `201`:

```json
{
  "data": {
    "id": "clx...",
    "username": "marc",
    "email": "marc@example.com",
    "role": "ADMIN"
  },
  "error": null
}
```

## Login

```
POST /api/auth/login
```

**Rate Limit:** 5 requests / 15 minutes / IP

**Request Body:**

```json
{
  "identifier": "marc",
  "password": "sicheres-passwort"
}
```

The `identifier` field accepts either a username or an email address.

```bash
curl -s -c cookies.txt https://your-instance.example.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"identifier": "marc", "password": "sicheres-passwort"}'
```

## Logout

```
POST /api/auth/logout
```

**Auth:** Session Cookie

Destroys the current session.

---

## Current User

```
GET /api/auth/me
```

**Auth:** Session Cookie or Bearer Token

Returns the current user including their Gravatar URL.

---

## Update Profile

```
PUT /api/auth/profile
```

**Auth:** Session Cookie

**Request Body:**

```json
{
  "email": "neu@example.com",
  "heightCm": 182,
  "dateOfBirth": "1990-05-15",
  "gender": "MALE"
}
```

All fields are optional. Only the provided fields are updated.

---

## Change Password

```
POST /api/auth/password
```

**Auth:** Session Cookie
**Rate Limit:** 5 requests / 15 minutes / user

**Request Body:**

```json
{
  "currentPassword": "altes-passwort",
  "newPassword": "neues-sicheres-passwort"
}
```

After a successful password change, all existing sessions are invalidated.

## Registration Status

```
GET /api/auth/registration-status
```

**Auth:** None

Checks whether registration is enabled (e.g. whether an admin already exists).

```json
{
  "data": {
    "registrationEnabled": true
  },
  "error": null
}
```

## Passkeys (WebAuthn)

HealthLog supports passwordless authentication via WebAuthn / Passkeys.

### Register a Passkey

```
POST /api/auth/passkey/register-options   — Retrieve options
POST /api/auth/passkey/register-verify    — Complete registration
```

### Passkey Login

```
POST /api/auth/passkey/login-options      — Retrieve challenge
POST /api/auth/passkey/login-verify       — Complete login
```

### Manage Passkeys

```
GET    /api/auth/passkeys                 — List all passkeys for the user
DELETE /api/auth/passkeys/[id]            — Remove a passkey
```

**Auth:** All passkey endpoints (except login) require an active session.
