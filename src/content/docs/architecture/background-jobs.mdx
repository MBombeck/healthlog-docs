HealthLog uses **pg-boss v12** for background jobs. pg-boss stores jobs directly in PostgreSQL — no separate message broker needed.

## Worker Startup

The worker starts automatically via the **Next.js Instrumentation API** (`register()`). On server startup, pg-boss is initialized and all cron jobs are registered. No manual startup is required.

The health check endpoint `/api/health` returns the current worker status.

## Job Overview

| Queue | Cron | Description |
|---|---|---|
| `medication-reminder-check` | `*/15 * * * *` | Phase-based medication reminders, snooze cleanup, Telegram message cleanup |
| `withings-fallback-sync` | `0 * * * *` | Fallback polling for Withings data (supplements webhook-based sync) |
| `moodlog-sync` | `30 * * * *` | MoodLog fallback sync (mood data) |
| `insights-general-status` | `0 2 * * *` | Calculate general health insights |
| `insights-blood-pressure-status` | `5 2 * * *` | Blood pressure insights |
| `insights-weight-status` | `10 2 * * *` | Weight insights |
| `insights-pulse-status` | `15 2 * * *` | Pulse insights |
| `insights-bmi-status` | `20 2 * * *` | BMI insights |
| `insights-medication-compliance-status` | `25 2 * * *` | Medication compliance insights |
| `insights-mood-status` | `30 2 * * *` | Mood insights (when MoodLog integration is active) |
| `data-backup` | `0 3 * * 0` | Weekly encrypted database backup (Sunday 03:00 UTC) |
| `telegram-message-cleanup` | On-demand | Deletes expired Telegram messages (triggered by other jobs) |

## Medication Reminders

The `medication-reminder-check` job runs every 15 minutes and checks for each user:

1. **Due reminders** — Scheduled intakes without confirmation, phase escalation (gentle, urgent, missed)
2. **Snooze expiry** — Expired snooze timers reactivate the reminder
3. **Message cleanup** — Deletes outdated Telegram reminder messages

Escalation phases and thresholds are controlled through `ReminderPhaseConfig` (per medication) and the admin settings (`reminderLateMinutes`, `reminderMissedMinutes`).

## Insights Jobs

The seven insights jobs run staggered between 02:00 and 02:30 UTC to avoid overloading the database with concurrent queries. Each job calculates trends, averages, and anomalies for its respective measurement type.

## Data Backup

The weekly backup job creates an encrypted export of all user data. Metadata is stored in the `DataBackup` model.

## Error Handling

pg-boss provides built-in retry behavior. Failed jobs are retried with exponential backoff. Persistent failures are logged and recorded in the audit log.
