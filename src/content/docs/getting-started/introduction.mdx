## What is HealthLog?

HealthLog is a **self-hosted, mobile-first Progressive Web App** (PWA) for comprehensive health tracking. It combines manual vital sign logging with automatic device syncing, smart medication management, and AI-powered insights.

All your data stays on your own infrastructure — no cloud lock-in, no subscriptions, full control.

## Key Features

- **Vital Sign Tracking** — Weight, blood pressure, heart rate, body fat, sleep, activity, mood
- **Medication Management** — Schedules, dosages, as-needed medications, compliance analytics
- **Automated Reminders** — Telegram, ntfy, Web Push notifications
- **Withings Sync** — Automatic import from smart scales and blood pressure monitors
- **AI Insights** — OpenAI integration (BYOK), trend analysis, personalized recommendations
- **Medical Reports as PDF** — Exportable reports with charts and historical data
- **Gamification** — 30+ achievements to reward consistent tracking
- **Passkey Authentication** — Passwordless login via WebAuthn
- **Dark-only Dracula Theme** — Easy on the eyes at any time of day

## Tech Stack

| Component | Technology |
|---|---|
| Framework | Next.js 16 (App Router, React Server Components) |
| Language | TypeScript (strict mode) |
| Database | PostgreSQL 16 |
| ORM | Prisma 7 with PrismaPg Adapter |
| Styling | Tailwind CSS 4 + shadcn/ui (Dracula Theme) |
| Client State | TanStack Query 5 |
| Auth | SimpleWebAuthn v13 (Passkeys) + session-based |
| Background Jobs | pg-boss v12 |
| Containerization | Docker + Docker Compose |

## Requirements

To run HealthLog, you need the following:

- **Docker** and **Docker Compose** (v2+)
- A **reverse proxy with HTTPS** (e.g., Nginx Proxy Manager, Traefik, Caddy)
- **PostgreSQL 16** (included in the bundled `docker-compose.yml`)
- A **domain with a valid SSL certificate** (Passkeys require HTTPS)

:::caution
Passkeys only work over HTTPS. Running HealthLog without a valid SSL certificate is not supported.
:::
