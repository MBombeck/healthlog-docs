## Prerequisites

- Docker Engine 24+ and Docker Compose v2+
- Reverse proxy with HTTPS (Nginx Proxy Manager, Traefik, Caddy, etc.)
- Domain with a valid SSL certificate
- At least 512 MB RAM and 1 GB disk space

## Step 1: Clone the Repository

```bash
git clone https://github.com/MBombeck/healthlog.git
cd healthlog
```

## Step 2: Generate Secrets

You need three cryptographic keys. Generate them with `openssl`:

```bash
# Session secret (cookie signing)
openssl rand -hex 32

# Encryption key (AES-256-GCM for sensitive database fields)
openssl rand -hex 32

# HMAC key (API token hashing)
openssl rand -hex 32
```

:::danger
The `ENCRYPTION_KEY` encrypts sensitive data in the database (tokens, API keys). If this key is lost or changed, all encrypted data becomes permanently unrecoverable. Store it in a safe place.
:::

## Step 3: Configuration

<Tabs>
<TabItem label="Docker Compose">

Create a `.env` file in the project directory:

```bash title=".env"
# Database
DATABASE_URL=postgresql://healthlog:YOUR_DB_PASSWORD@db:5432/healthlog

# Security (use the values from Step 2)
SESSION_SECRET=your_session_secret_here
ENCRYPTION_KEY=your_encryption_key_here
API_TOKEN_HMAC_KEY=your_hmac_key_here

# App URL (must be HTTPS)
NEXT_PUBLIC_APP_URL=https://health.your-domain.com
```

The bundled `docker-compose.yml` includes PostgreSQL 16 and the HealthLog app:

```yaml title="docker-compose.yml"
services:
  app:
    image: ghcr.io/mbombeck/healthlog:latest
    restart: unless-stopped
    env_file: .env
    ports:
      - "3000:3000"
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_USER: healthlog
      POSTGRES_PASSWORD: YOUR_SECURE_DB_PASSWORD  # openssl rand -base64 24
      POSTGRES_DB: healthlog
    volumes:
      - healthlog_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U healthlog"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  healthlog_data:
```

</TabItem>
<TabItem label="Manual Installation">

To install manually without Docker:

```bash
# Node.js 22+ and PostgreSQL 16 must be installed
# pnpm 10+ must be installed (corepack enable)

# Install dependencies
pnpm install

# Generate Prisma Client
pnpm db:generate

# Run database migrations
pnpm db:migrate:deploy

# Create production build
pnpm build

# Start the app
pnpm start
```

Set the environment variables directly or via a `.env` file:

```bash title=".env"
DATABASE_URL=postgresql://healthlog:password@localhost:5432/healthlog
SESSION_SECRET=your_session_secret_here
ENCRYPTION_KEY=your_encryption_key_here
API_TOKEN_HMAC_KEY=your_hmac_key_here
NEXT_PUBLIC_APP_URL=https://health.your-domain.com
```

</TabItem>

## Step 4: Start the Application

```bash
docker compose up -d
```

Check the logs to confirm a successful startup:

```bash
docker compose logs -f app
```

The application is ready when you see:

```
Ready on http://0.0.0.0:3000
```

## Step 5: Set Up the Reverse Proxy

Configure your reverse proxy to forward HTTPS traffic to port 3000 of the HealthLog container.

:::caution
HTTPS is mandatory. Passkeys (WebAuthn) only work in a secure context. Without a valid SSL certificate, registration and login will not function.
:::

## Step 6: Register the First User

Open `https://health.your-domain.com` in your browser. The first user to register is automatically granted administrator privileges.

:::caution
Register the admin account **immediately** after installation, before the URL is publicly accessible. Then disable registration under **Settings > Administration** to prevent unauthorized account creation.
:::

:::danger
**Never** commit the `.env` file to a Git repository. Make sure `.env` is listed in `.gitignore`. It contains cryptographic keys and database credentials.
:::
